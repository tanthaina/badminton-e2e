# 🛠️ สรุปปัญหาและวิธีแก้ไข (MacroDroid + Firebase)

เอกสารนี้รวบรวมปัญหาที่เคยพบและวิธีแก้ไขระหว่างการเชื่อมต่อแอป **MacroDroid** (ดักฟังแจ้งเตือนโอนเงิน) เข้ากับ **Firebase Realtime Database** เพื่อทำงานร่วมกับโปรแกรม Voicepen

---

## 🛑 ปัญหาที่ 1: โปรแกรมเมินยอดเงินหลักพัน (ติดเครื่องหมายลูกน้ำ)
* **อาการ:** เวลายอดโอนมีหลักพันขึ้นไป เช่น `1,500.00` เมื่อส่งเข้าโปรแกรม โปรแกรมจะอ่านได้แค่ `1` บาทถ้วน
* **สาเหตุ:** คำสั่ง `parseFloat()` ใน JavaScript เมื่อเจอเครื่องหมายลูกน้ำ (`,`) จะหยุดอ่านค่าทันที
* **การแก้ไข:** ในไฟล์ `app.js` ได้เพิ่มโค้ดเพื่อดักลบเครื่องหมายลูกน้ำออกก่อนทำการแปลงเป็นตัวเลข:
  ```javascript
  let rawAmount = data.amount;
  if (typeof rawAmount === 'string') {
      rawAmount = rawAmount.replace(/,/g, '');
  }
  const amount = parseFloat(rawAmount);
  ```

## 🛑 ปัญหาที่ 2: โอน 1 บาท แต่ประวัติในเว็บขึ้น 2 บาท (หรือเลขเก่า)
* **อาการ:** ทดสอบโอนเงินจริง 1 บาท แต่หน้าเว็บดันมีป๊อปอัปและประวัติขึ้นเป็น 2 บาท (ซึ่งเป็นเลขที่เคยใช้ทดสอบก่อนหน้านี้)
* **สาเหตุ:** เกิดจาก **ข้อมูลทดสอบเก่าตกค้างอยู่ใน Firebase** 
  - หากหน้าเว็บปิดอยู่ตอนที่ MacroDroid ยิงข้อมูล ยอดนั้นจะค้างอยู่ใน Database
  - เมื่อเปิดหน้าเว็บและกด "เปิดรับยอดโอน LINE" คำสั่ง `child_added` ของ Firebase จะ **ดูดข้อมูลเก่าทั้งหมด** ที่ค้างอยู่ออกมาทำงานพร้อมกัน ทำให้เห็นยอดเก่าโผล่มาด้วย
* **การแก้ไข:** ไม่ต้องแก้โค้ด เพราะโปรแกรมมีคำสั่ง `snapshot.ref.remove()` ซึ่งจะทำการลบยอดเก่าทิ้งทันทีที่ดูดลงมาแล้ว (เมื่อดูดเสร็จ Firebase จะโล่ง การทดสอบครั้งต่อไปยอดจะเข้าปกติ)

## 🛑 ปัญหาที่ 3: ตั้งตัวแปรแล้ว แต่เงินไม่เข้าแอป (เงียบสนิท)
* **อาการ:** ใน MacroDroid ตั้งค่า HTTP Request เป็น `{"amount":"vtype=5"}` หรือพิมพ์ชื่อตัวแปรผิด ทำให้เงินไม่เข้าเว็บเลย
* **สาเหตุ:**
  1. พิมพ์รูปแบบตัวแปรผิด เช่น พิมพ์เป็น `vtype=` ซึ่ง MacroDroid ไม่รู้จัก ทำให้แอปส่งข้อความตรงๆ ไปแทนที่จะเป็นตัวเลข
  2. โปรแกรมเว็บดักไว้ว่า ถ้ารับค่ามาแล้วไม่ใช่ตัวเลข (เป็น Text) ให้ **เมินทิ้ง (Ignore)** ทันที
* **การแก้ไขที่ถูกต้องใน MacroDroid:** 
  - ต้องใช้คำนำหน้าตัวแปรให้ถูกประเภท:
    - `{lv=ชื่อตัวแปร}` (สำหรับ Local Variable / ตัวแปรท้องถิ่น) **<< ส่วนใหญ่ใช้ตัวนี้**
    - `{v=ชื่อตัวแปร}` (สำหรับ Global Variable / ตัวแปรส่วนกลาง)
  - รูปแบบ JSON ในกล่อง Body ที่ถูกต้องคือ:
    ```json
    {"amount": "{lv=transfer_amount}"}
    ```
  - *ทริค:* ควรใช้ปุ่ม `...` สีน้ำเงินใน MacroDroid เพื่อกดเลือกชื่อตัวแปร ระบบจะใส่ปีกกาที่ถูกต้องให้เอง เพื่อป้องกันการพิมพ์ผิด

## 🛑 ปัญหาที่ 4: ต้องการจำลองแจ้งเตือน แต่ทริกเกอร์ไม่ทำงาน
* **อาการ:** สร้างมาโครจำลองแจ้งเตือนขึ้นมา แต่มาโครตัวหลักที่ดักฟังไม่ยอมทำงาน
* **สาเหตุ:** มาโครตัวหลักถูกตั้งทริกเกอร์ไว้ว่า รับแจ้งเตือนเฉพาะจากแอป **LINE** เท่านั้น แต่แจ้งเตือนจำลองถูกส่งมาจากแอป **MacroDroid**
* **การแก้ไขชั่วคราว:** เข้าไปแก้ทริกเกอร์ในมาโครตัวหลัก ให้ติ๊กถูกเพิ่มแอป **MacroDroid** เข้าไปด้วย (เมื่อทดสอบเสร็จ ค่อยเอาติ๊กถูกออกเพื่อความปลอดภัย)
* **ทางเลือกที่ดีกว่า:** ให้เพื่อนเปลี่ยนชื่อโปรไฟล์ใน LINE เป็น `Krungthai Connext` แล้วพิมพ์ข้อความจำลองส่งมาให้ วิธีนี้จะเนียนและสมจริงที่สุด

---

> **💡 คำแนะนำสำหรับการพัฒนาต่อไป:** 
> หากสูตร Regex ของ MacroDroid ดึงข้อความไม่สำเร็จ ตัวแปรจะไม่มีการอัปเดตและจะเก็บค่าเก่าเอาไว้ ควรตั้งเงื่อนไข (Condition) ใน HTTP Request ว่า *ให้ส่งข้อมูลเมื่อมีการสกัดตัวเลขสำเร็จเท่านั้น* เพื่อป้องกันการส่งยอดเก่าซ้ำครับ
